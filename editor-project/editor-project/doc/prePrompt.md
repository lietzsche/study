### 🎯 프로젝트 개요

**프로젝트 이름:** TDD 기반 문서 편집기 (Text Editor)
**언어:** Java (JavaFX 기반 GUI)
**목표:** 테스트 주도 개발(Test-Driven Development) 방식으로 실제 사용 가능한 문서 편집기 구현
**사용 목적:** 직접 글을 작성하고 관리하기 위한 개인용 에디터 개발

---

### 🧩 개발 방식 및 흐름

1. **TDD(테스트 주도 개발)** 원칙 적용

    * RED: 실패하는 테스트 작성
    * GREEN: 최소한으로 테스트 통과
    * REFACTOR: 중복 제거 및 구조 개선

2. **프로토타입 → 확장 구조 설계 → 세부 기능 추가** 순서로 진행

    * 프로토타입은 MVP(Minimum Viable Product) 형태로 간단한 문서 작성/저장 중심
    * 이후 기능 단위로 테스트 케이스 및 구현 추가

---

### 📚 커리큘럼 (단계별 학습 및 구현)

#### **1단계. 환경 세팅 및 기본 구조**

* [이론] JUnit5로 TDD 기본 사이클 실습
* [실습] Gradle/Maven 기반 프로젝트 세팅
* [실습] JavaFX 기본 창 띄우기 (`Stage`, `Scene`, `TextArea`)

#### **2단계. 텍스트 입력 기능 (프로토타입 1)**

* [이론] MVC 패턴과 TDD 적용 방식
* [실습] `TextArea`로 텍스트 입력/표시
* [TDD] `Document` 클래스 작성 및 테스트 (텍스트 추가, 삭제, 조회)

#### **3단계. 파일 저장 및 불러오기 기능**

* [이론] IOStream, FileChooser 이해
* [실습] `FileService` 구현 (`save`, `load` 메서드)
* [TDD] `FileServiceTest` 작성 (mock 파일 테스트 포함)

#### **4단계. Undo / Redo 기능 구현**

* [이론] Command 패턴 + 상태 저장 원리
* [실습] `UndoManager` 작성
* [TDD] 명령 단위로 테스트 작성 (`insert`, `delete`, `undo`, `redo`)

#### **5단계. 자동 저장 및 복구 기능**

* [이론] 스레드 / ScheduledExecutorService 개념
* [실습] 주기적 임시 저장 구현
* [TDD] `AutoSaveService` 테스트 작성 (MockTime 기반)

#### **6단계. UI 개선 및 편의 기능 추가**

* [이론] JavaFX 이벤트 및 바인딩
* [실습] 메뉴바, 단축키, 상태바 추가
* [TDD] UI 로직 분리 및 Controller 테스트

#### **7단계. 서식 기능 (폰트, 크기, 색상)**

* [이론] 스타일 관리, RichTextFX 라이브러리 연동
* [실습] 폰트 변경, 색상 변경 기능 추가
* [TDD] 서식 관련 모델 단위 테스트 작성

#### **8단계. 고급 기능 확장 (선택)**

* 다중 탭 문서 지원
* Markdown 프리뷰 기능
* 클라우드 백업 (REST API 연동)
